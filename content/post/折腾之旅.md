## 前言

二零一九年过得飞快, 新的一年突如其来, 各种大事件防不胜防. 这一年注定被历史所记忆.

本来想点东西, 记录下过去的二零一九, 但是临到写时, 却感觉无从下手. 想写的太多了. 便只好又将时间拉长, 准备直接回顾下整个本科学习和成长, 但是唠唠叨叨, 感觉一时半会儿根本写不完. 最后还是想着先把自成体系的部分先抽离出来, 弄成独立的文章, 一部分一部分的写出来, 或许会更容易些. 毕竟阶段性的奖励带来的持续的快乐, 要比最终一次性的大礼包造成的瞬间幸福, 要来的更实在, 更享受些.

本文主要回顾记录属于自己的一些经历. 希望成为自己日后回忆的一些素材. 外加上, 心里总是存在那个侥幸: 或许对别人有用呢? :laughing: 我很明白现在自己的水平, 编程能力只能说是一般. 掌握的技巧, 实战的经验, 都是少的可怜. 但是至少, 我已经可以使用自己熟悉的语言, 做一些自己想做的事了. 在我看来, 这已经算是一种自我水平的认证了. 

我写的这些经历, 都是从一个普普通通的喜欢探索, 热爱技术的人的角度出发, 从现在审视过去, 只希望自己走过的弯路, 遇到的挫折, 可以成为自己日后成长的养分. 至于能不能对他人有用, 这就管不着了. 我更多是想说, **一切出于热爱.** 你若真的热爱, 你自会成长.

## 缘起

最开始对编程感兴趣, 还是来自于高中同学的一些夸张的表述. 虽然这些描述在现在看来, 倒让我觉得他有可能是在胡扯. 但是不能不说, 他让我对于这一极具创造性的行业充满了向往. 外加上那时家里也终于有了电脑, 想法与物质都满足的情况下, 自然就要开始活动了. 这也算是揭开了我的折腾之旅. 可惜的是, 那时并不懂专门去学习编程 (现在想想, 如果在当时有一个合适的引路人, 那估计我在多年后的现在可能会和当前由很大的不同. 可惜没有如果), 而是走上了另外一条偏路, 也是当时最直接和简单的一条路, 折腾电脑里的各种软件.

记得在当时, 心理上的洁癖就开始有所显现, 特别喜欢删除电脑里的各种无用的文件和文件夹, 有时候导致电脑不得不重装系统. 记得那时用的还是ghost一键还原. 但是还原的次数多了, 反倒把硬盘给弄坏了. 也随之有了我人生中第一次去电脑城的经历. 自己抱着个主机, 坐着公交车, 成为了某种程度上的 "众人关注的焦点", 还是蛮尴尬的, 当然这些都是后话. 

除了文件系统上的洁癖 (虽然这一点在现在都仍然没有改变. 尤其受不了混乱的文件放置方式. 但是有时候就得逼着自己去接受, 因为整理的过程太费时间. 当然, 安卓手机文件管理中的散乱的零碎文件我是不会去管的, 主要是它们太多了, 又无法知道它们的用处), 各种软件的尝试也成了这段时期内, 我使用电脑 "从事" 的主要工作.

开始的时候家里没有网, 就专门去网吧下载各种软件, ^_^. 后来家里有网了, 就更是各种折腾. 这段经历, 虽然没有增长什么实际的能力, 但是倒让我对于各大杀毒软件, 系统清理软件, 音乐播放器, 视频播放器, 图片查看器, 浏览器等等, 都尝试过. 期间也见识过各大 "毒瘤" 们的手段. 所以现在我对于各家的产品很挑剔, 也逐渐培养起了一些使用windows的习惯, 同时也对windows系统的一些设置和工具更加熟悉. 这些都是最基础的一些习惯, 似乎对于后期的编程学习没有太多的帮助, 但实际上却也为后期的学习铺平了道路. 

**现在的一切都是未来成长的铺路石**.

## 初识

虽然后来对于编程也开始了各种粗浅的尝试 (例如, 安装环境, 这真是一个**恒久**的话题了), 但是真正意义上开始接触编程还是在大学. 大一的时候参加了一个社团. 它比较偏重于培养技术, 包括硬件 (单片机, 焊板子等) 和软件 (重点在于编程和算法等) 两个组. 

当时出于对编程的兴趣, 我选择进了软件组. 但这个 "进入" 指的是有了留下来的机会, 并不代表最后真的可以留下来. 社团里的学长给我们每个人都布置了一个阶段性的考核任务, 只有完成这个任务, 才可以留下来. 

我的任务是学习C语言, 用其写一个贪吃蛇小游戏. 这对于当时毫无经验, 从零入门C语言的我, 真的是一个非常具有挑战性的任务, 折腾了很久, 最终也还是选择了放弃, 选择了主动离开. 当时有很多的考量. 

* 一来是对于这个任务的不自信. 当时其实是有点**迷茫**的, **感觉自己像无头苍蝇一样, 没有方向, 没有规划, 内心很焦虑, 很难受**. 现在看来, 其实更多还是缺少编程思想的锻炼, 缺乏必要的流程分解, 分而治之的能力. 
* 二来是因为当时又参加了一个机器人社团. 我觉得这个似乎更有意思. **新知识的学习, 一味的靠所谓意志力的驱动, 是很难持久的**. 太难了, 因为人总是懒惰的且贪图舒适的, 想要强迫自己? 那是不可能的.

## 兴趣

新社团的加入, 实际上真正让我有了编程的欲望和动力, 这在很大程度上得益于此时终于有了一位引路的学长. 这个社团不同于之前的, 这里的气氛更加轻松, 活跃. 我们有着一位 "手把手" 教我们写代码的学长. 这在初期的学习中, 给我带来了非常大的帮助. 在这里对他表示感谢. 这位学长教我们 C 和一些简单的 C++ 的知识. 

这段时间主要是在学习 OpenCV 的一些操作, 这和我们在后期学习中, 处理机器人的视觉, 是有着很大的关系的. 这个时候我也算是开始接触计算机视觉了. 话说起来, 在当时, 我根本就没有想到, 自己硕士的方向, 也会是计算机视觉. 

在我们熟悉了 C 和基本的 OpenCV 的使用之后, 我们又开始接触了 Python, 这也是我当前的主力语言. Python 以其简洁有力的编程风格瞬时吸引了我的兴趣. 那个暑假, 我从图书馆搞来一本讲解 Python 2.7 的语法书, 带回了家. 可是我高看了自己的毅力. 这类讲解编程语言语法细节的书本就枯燥, 外加上这书写的确实一般, 更加让我看不下去. 最后还是从网上找了个教学视频, 跟着学习. 视频的学习有几个好处: 一个是直观, 可以及时的看到代码的效果, 外加上视频录制者有时写代码的小错误, 我们可以看到, 可以引以为戒. 外加上视频教课的本身受限于时长, 往往会省略一些东西, 而这些东西对于初学者来说可能并不是最重要的. 所以整体来说, **看视频学习对于初学者而言是个不错的选择**. 我还记得, 这个视频里作者使用 CentOS 来作为开发环境. 似乎也是从这里, 我开始了解了 windows 之外的系统. 

想想现在, 我在 Ubuntu 上使用 Python 和 PyTorch 进行网络的搭建, 训练和测试, 这些和那段时间的学习, 似乎有着某些必然的联系.

## 尝试

再往后, 就是不断积累的过程了. 主动或者被动的接触了许多其他的编程语言:

* MATLAB: 主要是用在一些信号处理的实验中, 包括一般的信号处理和数字图像处理. 虽然 MATLAB 功能极其强大, 但是其自身动辄近十个G的体量, 让我望而却步. 再加上其可怕的内存消耗, 我的小本本根本扛不住. 很不愿意去尝试这个东西.
* LabVIEW: 当时主要是为了配合 Arduino 来实现特定的任务, 虽然最终没有做出想要的功能, :cry:. 不过感觉这门语言 (实际上更应该说是一套软件), 在工业生产中应该很有用处. 官方创造的生态环境也比较丰富和完整.
* 汇编语言: 主要是两门课, 一门是专门学习 8086 芯片, 捎带学习对应的汇编语言. 另一门则是学习 AT89C51 单片机. 当时在 C51 单片机上, 使用汇编语言来编写小程序 (一天天的, 光翻手册了:smile:). 印象比较深的是, 当时终于实现了在 12864 上绘制动图的小 "梦想" (之前看做硬件的同学经常聊这个, 感觉很有意思, 但是一直没有学会, 当时手头仅有显示静态图的例子, 后来查了好多资料才搞明白刷新的机制和方法).
* Verilog HDL: 这是在学习数字电路的时候接触的. 总的来说, 实际上是接触了两次, 一次是做数字电路实验, 另一次通信原理实验. 前者还亲自上手写了代码, 后者直接就是连框框, 调用各种仿真元件, 之后再烧到FPGA上, 课程内容折腾的一批, 感觉毫无收获.
* R: 这次 "冲动" 实际上是出于对其在数据挖掘领域的强大能力而做的一次尝试. 为了学习这门语言, 还专门在 edx 上找了一门入门教程, 看完了, 了解了基本语法, 然后就没有然后了. 
* Java: 这个倒是投入了不少的时间. 看了半本 *疯狂Java讲义*, 但是最终还是陷入了语法的泥潭, 抬头发现, 起点都还没到呢, 所以后来也就搁置了.
* 其他的, 例如 elisp, haskell, schema 等等, 都是浅尝辄止. 更多的是一时的冲动.

另外也接触了各种开发工具, 这又是一笔糊涂账了, 说不清了:expressionless:. 

## 踏实

现在看来, 这段时间实际上自己还是 **太浮躁** 了, 没有做到真正的静下心去深入钻研, 没有做到编程学习最需要的 "实践". **杂而不精, 自受其苦**. 所以现在, 我也不愿意再去做那些费时间, 却于我而言并无真正益处的事情, 多出的时间倒还不如更加深入的学习下 Python 和 PyTorch. 毕竟, 这个对于我来说, 是更重要的工具和知识.
